# 构图模式与原创规则

## Structure Types

Choose one structure. Do not mix too many.

### Workflow

Use for input -> process -> output, content production, automation, and AI workflows.

Draw messy input on one side, a handmade processing object or workbench in the middle, and clear output on the other side. Let the IP boy operate the key step.

### System Slice

Use for sources, filters, databases, rendering, assistant systems, or local components.

Draw only 3-5 key modules. Place the IP boy inside one module or between modules doing the important action.

### Before / After

Use for chaotic/orderly, manual/automated, scattered/collected, stuck/running.

Draw two states with a simple transition. Let the IP boy cause the transition instead of only pointing at it.

### Role State

Use for user pain, creator state, tool overload, information anxiety, or emotional shifts.

Draw 2-4 small scenes. Keep each scene to one action and one short label.

### Concept Metaphor

Use for abstract ideas such as content engine, trust building, feedback loops, learning routes, or decision systems.

Turn the concept into a low-tech object: box, desk, drawer, bridge, door, cart, sorter, pulley, conveyor, ladder, mailbox, well, press, or small handmade machine.

### Layered Method

Use for frameworks, capability stacks, or methodology layers.

Use stacked boxes, shelves, drawers, or steps rather than a formal pyramid. Let the IP boy assemble or arrange layers.

### Map Route

Use for user journeys, learning paths, launch paths, or handoff paths.

Draw a winding route with a few nodes. Let the IP boy walk, pull, mark, open gates, or connect the route.

### Small Comic Sequence

Use for process stories, failed-to-working changes, or before/after usage.

Use 2-4 panels. Keep each panel simple. Preserve the IP boy's visual traits in every panel.

## Metaphor Method

1. Convert the abstract idea into a physical action: carry, sort, connect, weigh, open, repair, fold, filter, collect, press, return, or hand off.
2. Convert the system into a low-tech object: workbench, box, drawer, bridge, gate, cart, pulley, basket, shelf, machine, path, or desk.
3. Make the IP boy perform the action: he should be inside the metaphor, using his body and hands to move the idea forward.

## Action Pool

- Carry, push, pull, sort, catch, pass, open, close, connect, repair, assemble, label, test, weigh, fold, stack, sweep, hold, bridge, pull a cord, operate a lever, arrange cards.

## Anti-Copy Rules

Do not reuse the original xiaohei-specific old compositions by default:

- Small black creature in a machine pulling a lever.
- Black creature as a funnel.
- Black creature cutting a material fish.
- Black creature pulling a handoff path.
- Black creature holding a pit warning sign.

For this IP, invent fresh physical scenes that fit a friendly creator boy: desk, workbench, boxes, doors, cards, shelves, paths, small handmade machines, and everyday tools.

Do not copy the exact reference image pose by default. Use the reference for character identity, not composition.
